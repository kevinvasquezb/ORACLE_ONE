Flexbox CSS: Guia Completo, Elementos y Ejemplos

juliana-amoasei
02/09/2021

Flexbox tiene como meta ser una forma ms eficiente de crear diseos, alinear y distribuir espacios entre tems en un contenedor, incluso cuando las dimensiones de esos tems son desconocidas y/o dinmicas (en virtud de eso el trmino "flex").

Aprendamos los fundamentos de CSS Flexbox para la alineacin y el posicionamiento, y cmo usar sus funcionalidades correctamente.

Qu es el Flexbox

Durante mucho tiempo, las nicas herramientas disponibles para crear diseos CSS y posicionar elementos con buena compatibilidad entre browsers eran float y position. Sin embargo, estas herramientas tienen algunas limitaciones muy frustrantes, especialmente cuando se trata de responsividad. Algunas tareas que consideramos bsicas en un diseo, como centrar verticalmente un elemento secundario en relacin con un elemento principal o hacer que los elementos secundarios ocupen la misma cantidad de espacio, o que las columnas tengan el mismo tamao independientemente de la cantidad de contenido interno, fueron imposibles. o muy difcil de manejar con floats o position, al menos de forma prctica y flexible. La herramienta Flexbox (de Flexible Box) fue creada para hacer estas tareas ms sencillas y funcionales: los secundarios de un elemento con Flexbox se pueden posicionar en cualquier direccin y pueden tener dimensiones flexibles para adaptarse.

Elementos

Flexbox es un mdulo completo y no una propiedad nica; algunos de ellos deben declararse en el contenedor (el elemento principal, que llamamos de flex container), mientras que otros deben declararse en los elementos secundarios (el flex tems). Si el diseo "estndar" se basa en las direcciones block e inline, el diseo Flex se basa en direcciones de "flex flow". A continuacin se muestra un diagrama de la especificacin, que explica la idea central detrs del diseo Flex.

Los tems se distribuirn en el diseo siguiendo el eje principal o transversal.

  • Eje principal: el eje principal de un flex container es el eje primario y a lo largo de l son insertados los flex tems. Precaucin: El eje principal no es necesariamente horizontal; Depender de la propiedad flex-direction (vea abajo).

  • Main-start | main-end: los flex tems se insertan en el contenedor empezando por el lado start, dirigindose hacia el lado end.

  • Tamao principal: El ancho o alto de un flex tem, dependiendo de la direccin del contenedor, es el tamao principal del tem. La propiedad de tamao principal de un flex tem puede ser tanto width como height, dependiendo de cul est en la direccin principal.

  • Eje transversal: El eje perpendicular al eje principal se llama eje transversal. Su direccin depende de la direccin del eje principal.

  • Cross-start | cross-end: Lneas flex se llenan con tems y se agregan al contenedor, comenzando desde el lado cross start del flex container hacia el lado cross end.

  • Cross size: El ancho o alto de un flex tem, dependiendo de lo que haya en la dimensin transversal, es el cross size del tem. La propiedad cross size puede ser el ancho o el alto del tem, lo que se encuentre en la transversal.

    Flex container es el elemento que involucra su estructura. Tu defines que un elemento es un Flex Container con la propiedad display y valores flex o inline-flex.

    <div class="flex-container">

    <div>1</div>

    <div>2</div>

    <div>3</div>

    </div>

    .flex-container {

    display: flex;

    }

    Flex tem son elementos secundarios del flex container.

    Ejes o Axes son las dos direcciones bsicas que existen en un Flex Container: main axis, o eje principal, y cross axis, o eje transversal.

    Propiedades del elemento principal

    Cuando usamos el Flexbox, es muy importante saber qu propiedades se declaran en el elemento principal (por ejemplo, una div que contendr los elementos a alinear) y cules se declararn en los elementos secundarios. A continuacin, se muestran las propiedades que deben declararse utilizando el elemento principal como selector (para alinear los elementos secundarios):

    Display

    Esta propiedad define un flex container; inline o block dependiendo de los valores pasados. Coloca todos los elementos secundarios directos en un contexto Flex.

    .container {

    display: flex; */\* or inline-flex \*/*

    }

    Tenga en cuenta que la propiedad CSS columns no tiene efecto en un flex container.

    Flex-direction

    Establece el eje principal, definiendo as la direccin en la que los flex tems estn alineados en el flex container. Flexbox es (con la excepcin de un wrapping opcional) un concepto de diseo unidireccional. Piensa en los flex tems inicialmente posicionaids o en lneas horizontales o en columnas verticales.

    .flex-container { flex-direction: row | row-reverse | column | column-reverse; }

    • row (estndar): de la izquierda a la derecha en ltr (left to right), de la derecha a la izquierda en rtl (right to left)

    • row-reverse: de la derecha a la izquierda en ltr, de la izquierda a la derecha en rtl

    • column: mismo que row, pero de arriba a abajo

    • column-reverse: mismo que row-reverse pero de abajo hacia arriba

    flex-wrap

    Por estndar, los flex tems todos intentarn encajarse en una sola lnea. Con esta propiedad puedes modificar este comportamiento y permitir que los tems pasen a la siguiente lnea segn sea necesario.

    .flex-container {

    flex-wrap: nowrap | wrap | wrap-reverse;

    }

    • nowrap (estndar): todos los flex tems estarn en una sola lnea

    • wrap: los flex tems se dividirn en mltiples lneas, de arriba a abajo

    • wrap-reverse: los flex tems se dividirn en mltiples lneas de abajo hacia arriba

    flex-flow

    • La propiedad flex-flow es una propiedad shorthand (una misma declaracin incluye varios valores relacionados con ms de una propiedad) que incluye flex-direction y flex-wrap. Determina cules sern los ejes principal y transversal del contenedor. El valor estndar es row nowrap.

    .flex-container { flex-flow: row nowrap | row wrap | column nowrap | column wrap; }

    Justify-content

    Esta propiedad define la alineacin de los tems a lo largo del eje principal. Ayuda a distribuir el espacio libre que queda en el contenedor, ya sea que todos los flex tems de una lnea sean inflexibles o flexibles, pero ya hayan alcanzado su tamao mximo. Tambin ejerce cierto control sobre la alineacin de los tems cuando sobrepasan el lmite de la lnea.

    .flex-container {

    justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;

    }

    • flex-start (estndar): los tems se alinean a lo largo del borde de inicio (start) de acuerdo con la flex-direction del contenedor.

    • Flex-end: los tems se alinean a lo largo del borde final (end) de acuerdo con la flex-direction del contenedor.

    • start: los tems se alinean a lo largo del borde de inicio de la direccin del writing-mode (modo de escritura).

    • end: los tems se alinean a lo largo del borde final de la direccin del writing-mode (modo de escritura).

    • left: los tems estn alineados a lo largo del borde izquierdo del contenedor, a menos que esto no tenga sentido con el flex-direction que se est utilizando. En este caso, se comporta como start.

    • right: los tems estn alineados a lo largo del borde derecho del contenedor, a menos que esto no tenga sentido con el flex-direction que se est utilizando. En este caso, se comporta como start.

    • center: los tems estn centrados en la lnea.

    • Space-between: los tems se distribuyen uniformemente a lo largo de la lnea; el primer tem al lado del borde inicial de la lnea, el ltimo tem al lado del borde final de la lnea.

    • Space-around: los tems se distribuyen en lnea con el mismo espacio entre ellos. Tenga en cuenta que visualmente el espacio puede no ser igual, ya que todos los tems tienen la misma cantidad de espacio en ambos lados: el primer tem solo tendr una unidad de espacio a lo largo del borde del contenedor, pero dos unidades de espacio entre l y el siguiente tem, porque el tem siguiente tambin tiene su propio espaciamiento que se est aplicando.

    • Space-evenly: los tems se distribuyen de manera que el espacio entre dos elementos cualesquiera en la lnea (incluso entre los tems y los bordes) sea igual.

    Nota: el soporte dado por los navegadores para estos valores es difuso. Por ejemplo, space-between no tiene soporte en ninguna versin de Edge (hasta la elaboracin de este tutorial) y start / end / left / right an no se han implementado en Chrome. Para obtener tablas detalladas, consulte MDN. Los valores ms seguros son flex-start, flex-end y center.

    Tambin hay dos palabras clave adicionales que puedes usar junto con estos valores: safe y unsafe. Safe asegura que, independiente de cmo hagas este tipo de posicionamiento, no sea posible "empujar" un elemento y hacer con que sea renderizado hacia afuera de la pantalla (por ejemplo, sobre el tope) de una manera que haga con que el contenido sea imposible de mover con el desplazamiento de la pantalla (el CSS llama a esto de "prdida de datos").

    Align-tems

    Establece el comportamiento estndar de cmo flex tems estn alineados segn el eje transversal (cross axis). En cierto modo, funciona de manera similar al justify-content, pero en el eje transversal (perpendicular al eje principal).

    . .flex-container {

    align-items: stretch | flex-start | flex-end | center | baseline;

    }

    • Stretch (estndar): estira los tems para llenar el contenedor, respetando el min-width/max-width).

    • Flex-start/start/self-start: tems se posicionan al inicio del eje transversal. La diferencia entre ellos es sutil y se refiere a las reglas de flex-direction o writing-mode.

    • center: tems se centran en el eje transversal.

    • baseline: tems se alinean de acuerdo con sus baselines.

    Los modificadores safe y unsafe se puede usar junto con todas estas palabras clave (por favor verifique el soporte de cada navegador) y sirven para evitar cualquier alineacin de elementos que haga con que el contenido sea inaccesible (por ejemplo, fuera de la pantalla).

    align-content

    Organiza las lneas dentro de un flex container cuando hay espacio adicional en el eje transversal, similar a cmo justify-content alinea tems individuales dentro del eje principal.

    Importante: Esta propiedad no tiene efecto cuando solo hay una lnea de flex tems en el contenedor.

    .flex-container {

    align-content: flex-start | flex-end | center | space-between | space-around | stretch;

    }

    • flex-start / start: tems alineados con el inicio del contenedor. El valor (con mayor soporte de los navegadores) flex-start se gua por la flex-direction, mientras que start se gua por la direccin del writing-mode.

    • Flex-end/end: tems alineados con el final del contenedor. El valor (con mayor soporte de los navegadores) flex-end se gua por la flex-direction, mientras que end se gua por la direccin del writing-mode.

    • center: tems centrados en el contenedor.

    • space-between: tems distribuidos uniformemente; la primera lnea al inicio del contenedor y la ltima lnea al final del contenedor.

    • space-around: tems distribuidos uniformemente con el mismo espaciamiento entre cada lnea.

    • space-evenly: elementos distribuidos uniformemente con el mismo espaciamiento entre ellos.

    • stretch (estndar): tems en cada lnea se estiran para ocupar el espacio remanente entre ellas.

    Los modificadores safe y unsafe se puede usar junto con todas estas palabras clave (por favor verifique el soporte de cada navegador) y sirven para evitar cualquier alineacin de elementos que haga con que el contenido sea inaccesible (por ejemplo, fuera de la pantalla).

    Propiedades de los elementos secundarios

    A continuacin, veremos propiedades que deben declararse teniendo como selector los elementos secundarios, es decir:

    1
    2
    3

    Esto significa que, donde hay un elemento principal con propiedad flex (o flex-container), tambin puede asignar propiedades flex especficas a elementos secundarios (flex-tem).

    Puede definir las propiedades a continuacin para solo uno de los elementos secundarios a travs de un identificador, como una clase especfica.

    order

    Determina el orden en que aparecern los elementos.

    Por estndar los flex tems, se organizan en la pantalla en orden de cdigo. Pero la propiedad order controla el orden en que aparecen en el contenedor.

    .flex-item {

    order: <nmero>; */\* el valor estndar es 0 \*/*

    }

    Flex-grow

    Define la habilidad de un flex tem de crecer, segn el caso. El valor de esta propiedad es un valor numrico sin indicacin de unidad, que se utiliza para calcular la proporcin. Este valor dicta la cantidad de espacio disponible en el contenedor que ocupar el tem.

    Si todos los tems tienen flex-grow establecido en 1, el espacio remanente en el contenedor se distribuir de manera uniforme entre todos. Si uno de los tems tiene el valor 2, ocupar el doble de espacio en el contenedor que los dems (o al menos intentar hacerlo).

    .flex-item {

    flex-grow: <numero>; */\* el valor default(estndar) es 0 \*/*

    }

    La propiedad no acepta valores negativos.

    flex-shrink

    Establece la habilidad de un flex tem para contraerse, segn el caso. .flex-item {

    flex-shrink: <nmero>; */\* el valor estndar es 0 \*/* }

    La propiedad no acepta valores negativos.

    flex-basis

    Establece el tamao estndar de un elemento antes de que se distribuya el espacio remanente del contenedor. Puede ser un largo (por ejemplo, 20%, 5rem, etc.) o una palabra clave. La palabra clave auto significa "observe mis propiedades de altura o ancho" (lo que era hecho por la palabra clave t main-size, que fue depreciada). La palabra clave content significa "establezca el tamao segn el contenido interno del tem"; esta palabra clave an no tiene mucho soporte, por lo que no es fcil de probar, al igual que sus relacionadas: max-content, min-content y fit-content.

    .flex-item {

    flex-basis: flex-basis: | auto; */\* el valor estndar es auto \*/*

    }

    Con el valor 0, el espacio adicional alrededor del contenido no se considera. Con el valor de auto, el espacio adicional se distribuye con base en el valor de flex-grox del tem.

    flex

    Esta es la propiedad shorthand para flex-grow, flex-shrink y flex-basis, combinadas. El segundo y tercer parmetros (flex-shrink y flex-basis) son opcionales. El estndar es 0 1 auto, pero si lo defines con un solo nmero, es equivalente a 0 1.

    .item {

    flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]

    }

    Se recomienda que utilices esta propiedad shorthand en lugar de definir cada una de las propiedades por separado. Shorthand establece los dems valores de forma inteligente.

    align-self

    Permite que la alineacin estndar (o lo que est definido por align-items) se sobrescriba para tems individuales.

    Por favor consulte la explicacin de la propiedad align-items para entender cules son los posibles valores.

    . .item {

    align-self: auto | flex-start | flex-end | center | baseline | stretch;

    }

    Importante!

    • CSS solo ve la jerarqua de principal-secundario; no aplicar propiedades Flex a elementos que no estn directamente relacionados;

    • Para que las propiedades funcionen en elementos secundarios, los principales deben tener propiedad display: flex;.

    • Las propiedades float, clear y vertical-align no tienen ningn efecto en flex-items.
